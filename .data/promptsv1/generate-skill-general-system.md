# General System Skill Generator

**Disambiguation**: This generates **system-level meta-skills** — cross-cutting rules for the skill system itself. For the manual skill-authoring guide, see [`generate-skill-general.md`](generate-skill-general.md).

> **Base instructions**: Read [\_base-skill-generator.md](_base-skill-generator.md) first for shared structure, quality gates, and execution order. Below are general-system-specific overrides.

```
ROLE:     Senior AI tooling engineer analyzing a project's skill system and cross-cutting conventions
GOAL:     Generate a general/meta skill that defines how to interact with the skill system itself
SCOPE:    Cross-cutting rules that apply regardless of which domain skill is active
OUTPUT:   SKILL.md + INJECT.md + LEARNED.md + references/
```

---

## PHASE 1: PROJECT SCAN — Skill System Analysis

Scan the project's skill infrastructure:

**Skill Directory Structure**

- List all skills in `.data/skills/`
- Read each skill's SKILL.md frontmatter (name, description, scope)
- Identify shared patterns across all skills (LEARNED.md, INJECT.md, references/, scripts/)
- Note which files are present vs missing in each skill

**Skill File Conventions**

- SKILL.md: structure, sections, frontmatter format, token budget
- INJECT.md: purpose, length, what goes in vs out
- LEARNED.md: sections (Corrections, Preferences, Discovered Conventions), format (`YYYY-MM-DD: rule`)
- references/: naming, organization, content depth
- scripts/: naming (validate-\*.sh), patterns

**Cross-Cutting Rules**

- Rules that appear in ALL skills (e.g., "write corrections to LEARNED.md")
- Rules about when to use SKILL.md vs LEARNED.md vs INJECT.md
- Rules about reading skill instructions before modifying skill files
- Rules about skill activation ("Announce skill usage")

**Project Tooling**

- How skills are generated (skillnir generate-skill)
- How skills are installed (skillnir install)
- How skills are synced (skillnir update)
- Prompt versions and scope system

## PHASE 2: SKILL.MD CONTENT — General System Focus

### When to Use Section

This skill activates when:

1. Working with ANY skill file (SKILL.md, LEARNED.md, INJECT.md, references/)
2. Adding corrections, preferences, or discovered conventions
3. Creating or modifying skill directory structure
4. Resolving ambiguity about where content belongs
5. Cross-skill coordination (rules that affect multiple domain skills)

### Do NOT Use Section

- Domain-specific code changes (use the appropriate domain skill)
- Infrastructure/CI/CD tasks (use devops skill)
- Application logic (use backend/frontend skill)

### Key Patterns Table

Must include these rules:

| Rule                     | Description                                                                            |
| ------------------------ | -------------------------------------------------------------------------------------- |
| Read before edit         | Always read a skill's SKILL.md before modifying any file in that skill's directory     |
| LEARNED.md for learnings | Corrections, preferences, and discovered conventions go to LEARNED.md, never SKILL.md  |
| SKILL.md is generated    | SKILL.md content is generated by the skill generator — manual edits may be overwritten |
| INJECT.md is a firewall  | INJECT.md stays under 150 tokens — only critical quick-reference facts                 |
| Announce activation      | Say "Using: [Skill Name] skill" at the start of any response using a skill             |
| Date format in LEARNED   | Use `YYYY-MM-DD: rule description` format in LEARNED.md entries                        |
| One rule per entry       | Each LEARNED.md entry should capture one discrete rule, not a paragraph                |

### Conventions Table

Must include file ownership rules:

| File                   | Purpose                                                | Who Edits                            |
| ---------------------- | ------------------------------------------------------ | ------------------------------------ |
| SKILL.md               | Generated skill definition — decision guide            | Skill generator only                 |
| LEARNED.md             | Session-accumulated corrections and preferences        | AI writes during sessions            |
| INJECT.md              | Always-loaded quick reference (hallucination firewall) | AI updates when hallucinations occur |
| references/\*.md       | Detailed documentation, code examples                  | Skill generator or AI when extending |
| scripts/validate-\*.sh | Convention validation scripts                          | Skill generator or DevOps            |

### Anti-Patterns Table

Must include:

| Anti-Pattern                                       | Why It's Wrong                                                     |
| -------------------------------------------------- | ------------------------------------------------------------------ |
| Writing preferences to SKILL.md                    | SKILL.md is generated — use LEARNED.md for session learnings       |
| Editing skill files without reading SKILL.md first | Miss instructions about where content belongs                      |
| Putting code examples in SKILL.md                  | SKILL.md budget is ≤300 lines — code goes in references/           |
| Skipping LEARNED.md check on ambiguity             | May repeat a question the user already answered in a prior session |
| Mixing multiple rules in one LEARNED.md entry      | Makes entries hard to scan and apply selectively                   |

## PHASE 3: REFERENCES

### references/skill-file-guide.md

Document the complete skill file system:

- Full directory layout with all possible files
- Detailed explanation of each file's purpose and constraints
- Token budgets (SKILL.md ≤300 lines/<3500 tokens, INJECT.md 50-150 tokens)
- Examples of good vs bad LEARNED.md entries

### references/cross-skill-rules.md

Document rules that span all skills:

- How skills interact with each other
- Priority when multiple skills could apply
- How to delegate between skills
